# 자주 묻는 질문

이것은 SideStore에 대한 자주 묻는 질문 목록입니다. 여기에는 SideStore 개발 과정에서 자주 받는 질문들과, 다양한 사용자들이 가질 수 있는 문제점들에 대한 답변이 포함되어 있어요.

## SideStore란 무엇인가요?

SideStore는 Apple ID만으로 iOS 기기에 앱을 사이드로드할 수 있게 해주는 iOS 애플리케이션이에요. SideStore는 개인 개발자 인증서로 앱을 재서명하고, [특별히 설계된 VPN](https://apps.apple.com/kr/app/localdevvpn/id6755608044)을 사용하여 iOS가 앱을 설치하도록 해요. SideStore는 7일 개발 기간이 만료되지 않도록 백그라운드에서 주기적으로 앱을 "새로고침"해요.

SideStore의 목표는 제한 없는 사이드로딩 경험을 제공하는 것이에요. 이것은 [AltStore](https://github.com/rileytestut/AltStore)의 커뮤니티 주도 포크 버전으로, 커뮤니티에서 가장 많이 요청된 기능 중 일부를 이미 구현 했습니다. 설치 과정에서 컴퓨터는 단 한 번만 필요합니다.

(기여를 환영합니다! 🙂)

### SideStore에는 어떤 한계점이 있나요?

만약 무료 Apple 계정을 사용한다면, SideStore는 한 번에 최대 3개의 앱만 설치할 수 있어요(SideStore 포함). 또한, 일주일에 설치할 수 있는 다른 앱은 10개로 제한돼요. (앱 ID라고 부름).

### JIT를 활성화할 수 있나요?

iOS 또는 iPadOS 16 이하를 사용하는 사용자는 SideStore의 'My Apps' 페이지에서 앱을 길게 누른 후 'JIT 활성화'를 선택하여 직접 JIT를 활성화할 수 있어요.

iOS 또는 iPadOS 17.4 이상을 사용하는 사용자는 SideStore 0.6.2 또는 StikDebug(구 StikJIT)와 같은 솔루션을 사용할 수 있어요. 자세한 내용은 [고급](advanced/jit) 섹션의 튜토리얼을 참고해주세요.

### SideStore는 언제 출시되나요?

SideStore는 현재 출시일이 정해져 있지 않으며, 공개 베타 상태에요.

### 지금 앱을 사이드로드할 수 있나요?

네, AltStore처럼 사이드로드가 가능합니다!

### AltStore (베타) 기능이 포함되어 있나요?

저희는 AltStore (베타) 기능을 구현하지만 항상 최신 버전은 아니에요. 최신 상태를 유지하려 노력하고 있으며, 필요한 경우 자체 방식으로 구현해요.

### VPN을 켜두면 배터리가 빨리 소모되나요?

아니요, 이는 자신의 기기에만 연결되기 때문입니다. 원격 서버를 사용하지 않으므로 외부 연결이 없어 추가 배터리 소모가 없어요.

### VPN이 사용자를 추적하나요?

아니요, 원격 서버를 사용하지 않기 때문에 추적하지 않아요.

### 3개 이상의 앱을 활성화 할 수 있나요?

일반적으로 불가능해요. 이는 iOS 및 개발자 계정 제한 사항 때문이에요. Apple은 무료 Apple 개발자 계정으로 설치할 수 있는 앱을 3개로 제한하기 떄문이에요.

이 제한을 해제할려면 (365일 만료 기간을 얻으려면) 연간 99달러(약 14만원)의 Apple 개발자 계정 요금을 지불할 수 있어요.

대안으로 LiveContainer를 사용할 수 있고, iOS 버전 18 db5/18.0.1 이하에서는 SparseRestore 익스플로잇을 활용하여 3개 앱 제한을 우회할 수 있어요. (10개 앱 ID 제한은 우회하지 못함)

이것의 사용법 튜토리얼은 [여기](advanced/alternative)에서 확인할 수 있어요.

### 문제나 버그가 있거나 기능이나 사용성 개선을 요청하고 싶습니다

[일반적인 문제](troubleshooting/common-issues)와 기존 GitHub 이슈를 확인해 주세요. 해당 사항을 찾지 못하셨다면, 저희 GitHub 페이지에 새로운 GitHub 이슈를 생성하여 버그 보고나 기능 요청을 해 주세요(적절한 라벨을 붙여 주세요)!

### 개발자인데, 이것이 저에게 어떤 영향을 미치나요?

SideStore와 호환되도록 어떠한 수정도 필요하지 않아요.

### AltStore를 설치해야 SideStore를 설치할 수 있나요?

아니요, 저희의 유용한 [가이드](installation/prerequisites)를 따라 SideStore를 직접 사이드로드할 수 있어요!

### 사용자로서 SideStore에 로그인하는 데 문제가 있습니다. 오류가 발생합니다

이런 현상이 발생하는 이유는 여러 가지가 있지만, 가장 일반적인 문제는 Anisette 서버의 일시적인 다운타임 때문이에요. 현재 기기의 설정 앱에서 SideStore의 "Anisette URL"에서 Anisette 서버를 변경할 수 있어요.

### SideStore에 스토어프론트가 있나요?

현재 커뮤니티 소스가 있어요! 소스 URL: https://community-apps.sidestore.io/sidecommunity.json

또한 개발자가 자체 앱을 게시할 수 있도록 지원하는 [SideStore Connect](https://connect.sidestore.io) 서비스도 운영 중이에요.

### 왜 내 앱에 "pacer test"라는 업데이트가 있나요?

이는 주요 버전 간의 문제가 있는지 확인하기 위한 테스트 업데이트이며, 일반적인 사용 중에는 보이지 않아야 해요. 출시 후에 이 문제가 보인다면 SideStore를 재설치하거나 버그로 보고해주세요.

### 왜 JIT 기반 앱이 자동으로 JIT를 활성화하지 않나요?

현재 자동 JIT 활성화 방법이 없으며, 설령 있다 하더라도 LocalDevVPN을 켜야 해요. 대안으로 StikDebug를 사용할 수 있어요.

### SideStore에 iPadOS UI가 있나요?

네, 있습니다!

### 저의 계정에 iMac이나 MacBook Pro가 추가된 것을 봤습니다. 안전한가요, 실제 Mac인가요?

이 Mac들은 실제가 아니며, 로그인을 돕기 위해 Anisette 서버에서 자동으로 생성된 Mac이에요. Anisette 서버로 전송되는 데이터는 SideStore에서만 사용되므로 개인 데이터는 전송되지 않아요.

### SideStore를 사용할 때 계정이 잠기는 이유는 무엇인가요?

많은 사용자가 사용하는 오래된 Anisette 서버는 Apple ID 잠금을 유발하는 것으로 알려져 있어요. 공식 Anisette 서버 중 하나를 사용하거나 자체 [anisette-v3-server](advanced/anisette)를 호스팅하세요.

### AltStore/Sideloady 앱을 가져올 수 있나요?

네, 기기에서 기존 앱을 삭제하지 마세요. SideStore로 들어가서 동일하거나 업데이트된 해당 앱의 IPA 파일을 사이드로드하면, 데이터는 그대로 유지된 채 SideStore 내 앱 목록에 추가될 것이에요.

### 로그인 후 가장 먼저 SideStore를 새로고침해야 하는 이유는 무엇인가요?

이는 iloader 및 기타 프로그램들이 종종 자체 앱 그룹 ID로 SideStore를 사이드로드하는 반면, 우리는 자체 그룹 ID를 사용하기 때문이에요. SideStore를 새로 고침하면 저희 그룹 ID로 변경되어 오래된 그룹 ID를 가진 앱들이 사라져요. 설치 후에는 SideStore를 새로 고침하기 전까지는 아무것도 사이드로드하지 않는 것이 가장 좋아요.

### SideStore가 OTA 업데이트를 지원하나요?

네! 업데이트 버튼이 떴을때 클릭하면 됩니다!

### SideStore 버전을 테스트하고 싶은데, 테스트 버전으로 전환할 때 데이터를 유지하고 싶습니다. 앱을 제거해야 하나요, 아니면 SideStore를 통해 테스트 버전을 사이드로드해야 하나요?

SideStore 또는 iloader를 통해 나이트리 빌드의 SideStore 버전을 사이드로드하면 데이터가 유지될거에요. 또는 베타 업데이트를 활성화하고 설정에서 "nightly"를 선택해주세요. 비공식 또는 나이트리 빌드를 테스트할 때 데이터 보관은 보장할 수 없어요.
