En utilisant JitterbugPair (*et pas* l'application Jitterbug) nous crééons un fichier d'appairage pour les applications, comme SideStore, permetant de communiquer avec votre appareil. C'est un prérequis pour utiliser SideStore, sinon ça ne fonctionnera pas.

Téléchargez JitterbugPair pour [Windows (64-bit)](https://github.com/osy/Jitterbug/releases/download/v1.3.1/jitterbugpair-win64.zip), [macOS](https://github.com/SideStore/SideStore-Docs/releases/download/need-a-place-to-put-jittterbug/jitterbugpair), ou [Linux](https://github.com/osy/Jitterbug/releases/download/v1.3.1/jitterbugpair-linux.zip).

## Instructions d'appairage

:::tip

Lorsque vous utilisez le stockage dans le cloud les extensions de fichier peuvent changer (habituellement en .txt). Il est dans ce cas recommandé de compresser en .zip avant le transfert en ligne. SideStore accepte uniquement les fichiers .mobiledevicepairing.
:::

### Windows

1. Extraire le fichier JitterbugPair.
2. Connectez votre appareil à votre ordinateur et déverrouillez votre appareil. Une fois bon, executez le programme.
    * Votre appareil peut vous demandez de faire confiance à votre PC. Veuillez à bien accepter car nécessaire pour le processus de pairage.
3. Une fois fini vous aurez un fichier `.mobiledevicepairing`.
4. Transfèrez ce fichier vers votre appareil à l'aide d'iTunes, ou tout autre méthode. Compresser le fichier avant de l'envoyer est la façon la plus sûre de ne pas le corrompre durant le transfert.
    * Lorsque vous utilisez le stockage dans le cloud les extensions de fichier peuvent changer (habituellement en .txt). Il est dans ce cas recommandé de compresser en .zip avant le transfert en ligne. SideStore accepte uniquement les fichiers .mobiledevicepairing. Il est également possible de de changer l'extension pour un `.plist`, compatible avec les anciennes versions de SideStore (0.1.1 et antérieur).

Sur Windows vous pouvez être ammené à excecuter le programme en utilisant l'invite de commande ou la Powershell dans le cas où le .exe ne fonctionne pas.

5. Dans l'explorateur de fichier trouver le dossier jitterbugpair et l'ouvrir.
6. Dans la barre de navigation où le fichier est localisé, clicquez dans un coin vide et tapez `powershell`. Cette action devrait ouvir (possiblement de couleur bleu) une fenêtre nommée Powershell.
7. À partir d'ici, tapez `./jitterbugpair.exe` et ensuite appuyez sur entrée tout en gradant votre appareil connecté et déverouillé


### macOS

1. **Extraire** `Jitterbugpair-macos.zip`.
2. **Crééez un mot de passe** pour votre appareil si vous ne l'avez pas déjà fait. déverrouillez votre appareil et connectez le à votre appareil via un câble. Si une requête apparaît, acceptez là.
3. Déverrouillez votre appareil.
4. Trouvez le fichier extrait `jitterbugpair` (il devrait avoir une icône verte et noire).
5. Ouvrez le Terminal (depuis Launchpad il est dans le fichier "Outils").
6. Déposez le fichier `jitterbugpair` dans le Terminal et appuyez sur "Return" ou Entrée.
7. If you get "macOS cannot verify that this app is free from malware":
   - Allez dans Réglages > Confidentialité et sécurité
   - Scrollez jusqu'à message à propos de l'app
   - Clicquez "Ouvrir"
   - Essayez de nouveau à excecuter le programme.
8. JitterBugPair va créé un **fichier d'appairage** avec l'extension `.mobiledevicepairing`.
9. Si vous ne trouvez pas le fichier d'appairage:
   - Copiez le nom du fichier créé
   - Collez le dans Finder
   - Si vous avez excecuté le programme plus d'une fois les fichiers devraient tous apparaître sous le même nom.
10. **Transférez le fichier d'apairage** vers votre appareil iOS en utilisant AirDrop, iCloud/One/Google Drive, un email, ou tout autre méthode. Pour un meilleur résultat, compressez le fichier en un .zip.

### Linux

These instructions expect that you are familiar with the linux commandline.

1. Extraire le fichier JitterbugPair, et ouvrir un Terminal (si vous ne l'avez pas déjà fait) au chemin du fichier extrait. 
2. Dans ce terminale entrer: run `chmod +x ./jitterbugpair`
3. Branchez votre appareil sur votre PC et dévereouillez sur le menu d'acceuil. Une fois fini, excecutez le programme dans votre Terminal avec `./jitterbugpair`
    *  Si vous avez une requête disant que que vous devez faire confiance à votre ordinateur, acceptez. Le cas échéant, rouvrez le programme.
4. Une fois fini vous aurez un fichier `.mobiledevicepairing` dans le fichier où vous avez ouvert jitterbugpair.
5. Transfèrez ce fichier vers votre appareil à l'aide d'iTunes, ou tout autre méthode. Compresser le fichier avant de l'envoyer est la façon la plus sûre de ne pas le corrompre durant le transfert.
    * Lorsque vous utilisez le stockage dans le cloud les extensions de fichier peuvent changer (habituellement en .txt). Il est dans ce cas recommandé de compresser en .zip avant le transfert en ligne. SideStore accepte uniquement les fichiers .mobiledevicepairing. Il est également possible de de changer l'extension pour un `.plist`, compatible avec les anciennes versions de SideStore (0.1.1 et antérieur).


Vous pourrez mieux réussir en excecutant simplement avec des permissions ou en ouvrant jitterbugpair depuis un manager de fichier, il est tout de même principalement recommandé de le faire depuis un Terminal.
