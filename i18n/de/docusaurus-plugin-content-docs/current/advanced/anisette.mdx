:::note
Wenn du eine SideStore-Version nach 0.4.0 verwendest, ist es unwahrscheinlich, dass dein Konto gesperrt wird, da zu diesem Zeitpunkt die Unterstützung für v3 Anisette-Server hinzugefügt wurde.

Daher ist es **sehr empfehlenswert**, einen v3-Server für die normale Nutzung von SideStore zu verwenden, wenn deine Version dies unterstützt.
:::

Anisette-Daten sind Informationen, die im App-Signierungsprozess verwendet werden und jedes Mal generiert werden müssen, wenn du Apps installierst oder aktualisierst. SideStore erhält diese Informationen von einem Server, der einen Mac nachahmt und dir die Anisette-Daten zurücksendet. Bei diesem Vorgang werden keine Kontoinformationen an den Server gesendet.

Wenn viele Personen denselben Anisette-Server verwenden (insbesondere einen v1-Server), werden die Sicherheitsvorkehrungen von Apple ausgelöst und die Konten, die diesen Rechner verwendet haben, gesperrt. Aus diesem Grund empfehlen wir die Verwendung einer separaten Brenner-Apple-ID anstelle deiner Haupt-ID.

Unser Team hat an besseren Ansätzen zur Vermeidung gesperrter Konten gearbeitet (in Form von _anisette-v3_). Wenn du jedoch eine ältere Version von SideStore verwendest, ist das Hosten deines eigenen anisette-Servers eine großartige Möglichkeit, um gesperrte Konten zu vermeiden. Dies ist unglaublich einfach, da unser aktueller Server quelloffen ist und für Docker-Umgebungen entwickelt wurde. Das bedeutet, dass wir einen Server mit nichts weiter als dem GitHub-Link bereitstellen können.

## Einsatz auf Render

[Render] (https://render.com/) ist ein Dienst zur einfachen Erstellung von Servern. Er unterstützt das Laden von Serverquellen von GitHub. Die kostenlose Version ist in Bezug auf die Ressourcen recht begrenzt, aber der Server kann ohne Probleme darauf laufen.

Da zu viele Personen dieselbe GitHub-URL verwenden, musst du das Git-Repository selbst forken.

1. [Registriere](https://github.com/signup) dich für ein GitHub-Konto, falls du noch keines hast.
2. [Öffne](https://github.com/Dadoum/anisette-v3-server) das Git-Repository.
3. Klicke oben rechts auf „Fork“ und forke das Repository zu deinem Konto.
4. Sobald das geforkte Repository erstellt ist, klicke auf das grüne Dropdown-Menü „Code“ oben rechts und kopiere die Git-URL.

* Merk dir das für später!

Sobald dies geschehen ist, kann der Render-Server erstellt werden.

1. [Registriere] (https://dashboard.render.com/register/) dich für ein Konto.
2. Erstelle einen [Neuen Webdienst](https://dashboard.render.com/select-repo?type=web).
3. Blättere nach unten zu **Öffentliches Git-Repository** und gebe die zuvor kopierte URL ein.
4. Klicke auf „Weiter“ und fülle das Formular aus:
    * **Name** - spielt keine Rolle
    * **Region** - Am nächsten bei dir
    * **Branchen** - Haupt (Standard)
    * **Root-Verzeichnis** - leer (Standard)
    * **Umgebung** - Docker (Standard)
    * **Instanztyp** - Gratis

5. Warte, bis der Server aufgebaut ist. Wenn er fertig ist, siehst du Folgendes:

```
Machine requires provisioning... done !
Your service is live 🎉
```

*Wenn der Build aus irgendeinem Grund fehlschlägt, klicke auf **Manuelle Bereitstellung** und dann auf **Letzte Übertragung bereitstellen**.

6. Suche den Link für deinen neuen Server oben auf der Seite:

```
https://[dein-server-name].onrender.com
```

## Erstellen einer benutzerdefinierten Anisette-Server-Liste

Seit der Veröffentlichung von Version 0.5.8 verwendet SideStore ein Anisette-Server-Listensystem, so dass offiziell unterstützte Anisette-Server leichter hinzugefügt werden können. Dies bedeutet, dass es ein wenig zusätzliche Arbeit zu tun gibt, um deinen eigenen Server zu verwenden, ohne die Aufnahme in die offizielle Liste zu beantragen.

Wenn du die Option zur Verwendung der enthaltenen Server beibehalten möchtest, führe die folgenden Schritte aus:
1. Gehe [hier] (https://servers.sidestore.io/servers.json), um die aktuelle Serverliste zu erhalten.
2. Gehe zu einem Filehosting-Dienst deiner Wahl, um die Datei im Internet zu hosten. Wir empfehlen die Verwendung von GitHub Pages, da dieser Dienst in Zukunft leichter zu bearbeiten ist und als Beispiel für diese Anleitung verwendet wird. Wenn du deine eigene Art des Hostings hast, passe die folgenden Schritte an dein Setup an.
3. Wenn du bereits eine Github Pages-Seite eingerichtet hast, fahre mit dem sechsten Teilschritt fort. Andernfalls gehe wie folgt vor:
* Erstelle ein neues GitHub-Repository entweder über die Plus-Schaltfläche (+) oben rechts auf dem Desktop, das Profilsymbol > Neu erstellen auf dem Handy oder [diese Seite](https://github.com/new).
* Lege den Namen [Dein GitHub-Benutzername].github.io fest, aktiviere „Dieses Repository mit einer README initialisieren“ und lege die übrigen Einstellungen nach deinen Wünschen fest.
* Wechsel nach der Erstellung auf die Registerkarte Einstellungen und navigiere zum Abschnitt Seiten unter Code und Automatisierung.
* Setze Build and deployment unter Quelle auf 'Deploy from branch'.
* Falls nicht automatisch ausgewählt, setze Branch auf 'main'.
* Kehre zur Registerkarte „Code“ zurück und klicke auf „Dateien hinzufügen“, dann auf „Dateien hochladen“ und wähle die zuvor heruntergeladene Datei „servers.json“ aus.
* Klicke nach dem Hochladen auf die Datei und klicke auf die Schaltfläche „Bearbeiten“.
* Scrolle bis zum letzten Server in der Liste und füge ein Komma nach der schließenden geschweiften Klammer ('}') des letzten Servers ein, und füge anschließend deinen eigenen Server im folgenden Format hinzu:
```
{
    "name": "[etwas Identifizierbares wie ein Benutzername]", //das spielt keine Rolle, es sei denn, du teilst diese Serverliste mit jemand anderem
    "address": "[die Url deines Servers]"
}
```

* Bestätige die Änderungen und gehe dann zu https://[Dein GitHub-Benutzername].github.io/[wo du die Datei abgelegt hast]/servers.json. Wenn die Datei geladen wird, kannst du mit dem nächsten Abschnitt fortfahren. Andernfalls musst du möglicherweise bis zu 10 Minuten warten, bis deine Website ordnungsgemäß für die Nutzung eingerichtet ist, bevor du es erneut versuchst.
Wenn du nur deinen eigenen Server verwenden möchtest, folge den obigen Schritten, aber anstatt die offizielle server.json zu benutzen, erstellen deine eigene mit dieser Formatierung:
```
{
    "servers": [
        {
            "name": "[wie oben]",
            "address": "[auch wie oben]"
        }
    ]
}
```
und fahre dann wie gewohnt fort.

## Verwende deinen Server mit SideStore

Um deine neu erstellte Serverliste in der SideStore-Anwendung zu verwenden, gehe wie folgt vor:

1. Öffne die Registerkarte Einstellungen in SideStore.
2. Scrolle nach unten und tippe auf "Anisette Server".
3. Tippe auf die URL der Liste und ersetze sie durch die URL deiner Serverliste.
4. Klicke auf 'Refresh Servers' und wähle deinen eigenen Server aus der aktualisierten Liste aus.

## Fertig!

SideStore verwendet nun deinen neuen benutzerdefinierten Anisette-Server.
